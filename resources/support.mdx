## Reporting Bugs

Help improve VulnTrack by reporting issues:

<Steps>
  <Step title="Search Existing Issues">
    Check if the bug has already been reported at [GitHub Issues](https://github.com/ogdmerlin/vulntrack/issues)
  </Step>

  <Step title="Verify on Latest Version">
    Confirm the bug exists in the most recent version of VulnTrack
  </Step>

  <Step title="Gather Information">
    Collect:
    - Steps to reproduce
    - Expected vs actual behavior
    - Environment details (OS, Node.js version, PostgreSQL version)
    - Screenshots or error messages
    - Relevant log output
  </Step>

  <Step title="Create Detailed Issue">
    Open a new issue with:
    - Clear, descriptive title
    - Complete reproduction steps
    - All relevant information
    - Code samples if applicable
  </Step>
</Steps>

<Info>
See the [FAQ](/resources/faq#troubleshooting) for common issues and solutions before reporting.
</Info>

## Getting Help

### Before Asking

1. **Read the documentation** - Most questions are answered here
2. **Check the FAQ** - Common issues and solutions
3. **Search existing discussions** - Someone may have asked already
4. **Review the changelog** - Recent updates may address your question

### Asking Good Questions

Get better answers by providing:

- **Clear problem description** - What are you trying to accomplish?
- **What you've tried** - Show your debugging steps
- **Environment details** - Version, OS, configuration
- **Code samples** - Minimal reproducible example
- **Error messages** - Complete, unmodified output

<Warning>
Never share sensitive information (passwords, API keys, vulnerability details) in public discussions.
</Warning>

## Security Issues

**Do not report security vulnerabilities through public GitHub issues.**

For security concerns:

1. **Email maintainers directly** - Contact information in GitHub profile
2. **Provide detailed information** - Steps to reproduce, impact assessment
3. **Allow time for response** - Give maintainers time to address issue
4. **Coordinate disclosure** - Work together on responsible disclosure

<Warning>
Public disclosure of security vulnerabilities before fixes are available puts all VulnTrack users at risk.
</Warning>

This is an open source project maintained by volunteers:

- **Critical bugs**: Best effort, usually within days
- **General issues**: Depends on maintainer availability
- **Feature requests**: May take weeks or months to implement
- **Documentation**: Often addressed quickly
